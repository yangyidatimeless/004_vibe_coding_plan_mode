## 🎨 前端交互设计建议

### 1. 计划展示界面

Plan 模式的核心是"透明化"，用户需要清晰地看到 AI 要做什么。我建议设计一个**分步计划卡片**：

```
┌─────────────────────────────────────────────────┐
│  📋 执行计划                                    │
├─────────────────────────────────────────────────┤
│  ✓ 1. 创建用户模型 (models/user.py)            │
│  ✓ 2. 创建认证服务 (services/auth.py)          │
│  ○ 3. 创建 API 端点 (routes/auth.py)           │
│  ○ 4. 添加单元测试 (tests/test_auth.py)        │
│  ○ 5. 更新依赖 (requirements.txt)              │
├─────────────────────────────────────────────────┤
│  [查看详情] [提出修改] [批准执行]               │
└─────────────────────────────────────────────────┘
```

**关键交互点**：
- 每一步前面用状态图标（✓ 完成 / ○ 待执行 / ⚠ 有问题）
- 点击某一步可以展开查看该步骤的详细操作
- 底部三个按钮：查看详情（展开所有细节）、提出修改（进入编辑模式）、批准执行（开始执行）

### 2. 执行过程可视化

执行过程中，用户需要实时看到进度：

```
┌─────────────────────────────────────────────────┐
│  🚀 执行中 - 步骤 3/5                           │
├─────────────────────────────────────────────────┤
│  ✓ 步骤 1: 创建用户模型 ... 完成 (2.3s)        │
│  ✓ 步骤 2: 创建认证服务 ... 完成 (1.8s)        │
│  ▶ 步骤 3: 创建 API 端点 ... 执行中            │
│    └─ 正在创建 routes/auth.py                  │
│    └─ 正在添加路由处理器                       │
│  ○ 步骤 4: 添加单元测试 ... 等待中             │
│  ○ 步骤 5: 更新依赖 ... 等待中                 │
├─────────────────────────────────────────────────┤
│  [暂停] [查看日志]                              │
└─────────────────────────────────────────────────┘
```

### 3. 异常处理交互

当某一步骤遇到问题时，需要清晰地告知用户并提供选项：

```
┌─────────────────────────────────────────────────┐
│  ⚠️ 需要你的确认                                │
├─────────────────────────────────────────────────┤
│  步骤 3 执行时遇到问题：                        │
│  文件 routes/auth.py 已存在，是否覆盖？         │
│                                                 │
│  [查看原文件] [创建新文件] [覆盖] [跳过此步]    │
└─────────────────────────────────────────────────┘
```

### 4. 响应式设计的考虑

- **桌面端**：可以展示完整的侧边栏计划面板
- **移动端**：折叠为可展开的卡片，优先展示当前步骤
- **深色模式**：需要适配代码高亮的对比度

### 5. 可访问性

- 每个状态图标都要有文字说明（screen reader 友好）
- 键盘导航支持（Tab 切换步骤，Enter 确认）
- 颜色不是唯一的状态标识（要配合图标和文字）

---

## 💡 我的建议

1. **先做 MVP**：第一步先实现最基本的计划展示 + 确认按钮，不要一开始就追求完美的可视化
2. **参考 Cursor**：可以直接体验一下 Cursor 的 Plan 模式，看看它的交互流程
3. **留出扩展空间**：组件设计时要考虑后续可能添加的功能（如步骤重试、回滚等）
