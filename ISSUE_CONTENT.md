## 📋 什么是 Plan 模式？

Plan 模式是现代 AI 编程助手（如 Cursor、OpenAI Codex）中的一个核心功能，它允许 AI 在执行代码修改之前，先生成一个详细的执行计划，让用户审查和确认。

## 🔍 Cursor 的 Plan 模式流程

### 1. **用户输入需求**
```
用户：帮我添加一个用户登录功能，需要支持邮箱和密码
```

### 2. **AI 分析代码库**
- 读取项目结构
- 理解现有代码风格
- 识别相关文件和依赖
- 检查是否有类似的实现模式

### 3. **生成执行计划**
AI 会输出一个结构化的计划，例如：
```markdown
## Plan

1. 创建用户模型 (models/user.py)
   - 添加 email 字段（唯一索引）
   - 添加 password_hash 字段
   - 添加 created_at 时间戳

2. 创建认证服务 (services/auth.py)
   - 实现 register() 方法
   - 实现 login() 方法
   - 实现 password hashing（使用 bcrypt）

3. 创建 API 端点 (routes/auth.py)
   - POST /api/register
   - POST /api/login
   - POST /api/logout

4. 添加单元测试 (tests/test_auth.py)
   - 测试注册流程
   - 测试登录流程
   - 测试密码加密

5. 更新依赖 (requirements.txt)
   - 添加 bcrypt
   - 添加 PyJWT

要我开始执行这个计划吗？
```

### 4. **用户审查与确认**
- 用户可以查看计划
- 可以提出修改意见
- 可以批准执行

### 5. **分步执行**
- AI 按步骤执行
- 每步显示进度
- 遇到问题会暂停并询问

### 6. **验证与提交**
- 运行测试
- 显示测试结果
- 生成 git diff
- 等待用户确认提交

## 🤖 OpenAI Codex 的 Plan 模式

Codex 的 Plan 模式更加自动化，主要特点：

### AGENTS.md 配置文件
```markdown
# AGENTS.md - 给 AI 的指令

## 项目结构
- src/ - 源代码
- tests/ - 测试文件
- docs/ - 文档

## 测试命令
- `npm test` - 运行所有测试
- `npm run lint` - 代码检查

## 代码风格
- 使用 TypeScript
- 遵循 ESLint 配置
- 函数使用箭头函数

## 提交规范
- 使用 Conventional Commits
- 每条提交要有清晰的描述
```

### Codex 的工作流程
1. **接收任务** → 用户在 ChatGPT 中输入需求
2. **沙箱环境** → 在隔离的云环境中加载代码库
3. **自主规划** → AI 自己制定执行计划
4. **执行任务** → 读取/编辑文件、运行命令、执行测试
5. **提供证据** → 显示终端日志、测试输出、引用来源
6. **提交结果** → 生成 PR 或直接集成

## 💡 关键差异

| 特性 | Cursor | Codex |
|------|--------|-------|
| 执行环境 | 本地 | 云端沙箱 |
| 计划可见性 | 显式展示计划 | 隐式规划 |
| 用户干预 | 每步可干预 | 异步执行 |
| 测试验证 | 本地运行 | 沙箱内运行 |
| 适用场景 | 实时协作 | 异步委托 |

## 🎯 我们可以借鉴的点

1. **显式计划生成** - 在执行前让用户知道要做什么
2. **分步确认机制** - 关键步骤需要用户批准
3. **测试先行** - 自动运行相关测试验证
4. **透明化** - 显示所有操作日志和证据
5. **可回滚** - 生成清晰的 diff，方便审查
